# TASK-072 - Unify toolbar UX, status colors, badges, and mode-aware device summary

## Контекст

У поточному UI є декілька неузгодженостей:

1. Filters (Active filters) використовують інший стиль, ніж badges у таблиці
2. Кольори статусів відрізняються між:
   - summary (Devices: 41 | 🟢12 ...)
   - рядками таблиці
3. Mode (End / All) не впливає на summary (Devices count)
4. Toolbar потребує уніфікації (TASK-071)

---

## Мета задачі

- уніфікувати візуальну мову UI
- синхронізувати кольори
- зробити filters частиною тієї ж системи badge
- зробити summary залежним від активного режиму (End / All)

---

## Що потрібно зробити

---

## 1. Filters = ті самі badges що і в таблиці

### Було:
```
[ 🔴 Offline ] [ 🎯 RANDOM ]
```

### Має бути:
```
[ OFFLINE ] [ RANDOM ]
```

### Вимоги:
- використовувати ТОЙ САМИЙ компонент badge
- ті самі кольори
- ті самі стилі
- додати:
  - hover
  - cursor pointer
  - active state

### Важливо:
НЕ створювати окремий стиль для filters

---

## 2. Уніфікація кольорів статусів

### Проблема:
різні кольори в:
- summary
- рядках таблиці

### Рішення:

Ввести єдині color tokens:

```
online
idle
unknown
offline
```

### Вимоги:
- всі компоненти використовують одні й ті самі кольори
- не допускається різниця між:
  - status dot
  - badge
  - summary

---

## 3. Devices summary залежить від Mode

### Нове правило:

Summary повинен показувати тільки ті дані, які відповідають поточному режиму:

---

### Mode = End

```
Devices: (кількість End devices)
🟢 🟡 🟣 🔴 → тільки для End devices
```

---

### Mode = All

```
Devices: (загальна кількість)
🟢 🟡 🟣 🔴 → для всіх пристроїв
```

---

### Вимоги:

- summary рахується з того ж dataset, який відображається в таблиці
- не використовувати глобальні значення
- summary має оновлюватись при:
  - зміні Mode
  - зміні filters
  - автооновленні

---

## 4. Toolbar layout (з урахуванням TASK-071)

Фінальний вигляд:

```
[ OFFLINE ] [ RANDOM ]   Clear ✕                    ⇅ ⟳   [ End | All ]   Devices: X | 🟢x 🟡x 🟣x 🔴x   Auto [ON]
```

---

## 5. Поведінка Mode

Mode має впливати на:

- список пристроїв
- summary
- (опціонально) filters

---

## Вимоги до логування

(англійською)

Додати:

- `Mode switched to END`
- `Mode switched to ALL`
- `Summary recalculated for current mode`

## Вимоги до тестування

### 1. Filters badges
- виглядають як у таблиці
- клікабельні
- коректно очищаються

### 2. Color consistency
- кольори однакові всюди

### 3. Mode → Summary
- End → показує тільки End devices
- All → показує всі

### 4. Dynamic update
- при зміні mode summary оновлюється
- при автооновленні значення змінюються

## Критерії приймання

- filters = ті самі badges що в таблиці
- кольори однакові по всьому UI
- summary залежить від Mode
- немає дублювання стилів
- UI виглядає консистентно
- всі значення коректні

---

## Очікуваний результат

UI стає:

- консистентним
- передбачуваним
- легким для читання

Toolbar працює як єдина система:

```
Filters → впливають на список → список визначає summary
Mode → впливає на список → список визначає summary
```
