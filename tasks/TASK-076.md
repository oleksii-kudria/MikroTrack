# TASK-076 - Align filters and device summary row to target layout

## Контекст

Потрібно виконати тільки одну зміну UI:

- привести строку з фільтрами
- та блок з даними по кількості девайсів

до вигляду, показаного на наданому зображенні.

Це вся задача.  
Без додаткових змін у рядках пристроїв, таблиці, status/time, badges у таблиці або інших частинах інтерфейсу.

---

## Що потрібно зробити

Привести верхню строку керування до такого вигляду:

```text
Clear ✕   |   Devices: 28 | 🟢 3 🟡 3 🟣 0 🔴 22   [ End | All ]   ⇅   ⟳   Auto [ON]
```

---

## Межі задачі

### Дозволено змінювати тільки:
- строку з active filters / clear
- блок Devices summary
- перемикач End / All
- кнопки sort / refresh
- toggle Auto
- відступи, порядок і вирівнювання цих елементів

### Не потрібно змінювати:
- структуру рядків пристроїв
- таблицю
- картки / badges у рядках
- логіку stale
- логіку state/timer
- інші блоки сторінки

---

## Цільовий порядок елементів

Порядок зліва направо має бути таким:

1. `Clear ✕`
2. divider
3. `Devices: X | 🟢 x 🟡 x 🟣 x 🔴 x`
4. `[ End | All ]`
5. `⇅`
6. `⟳`
7. `Auto [ON]`

---

## Вимоги до layout

- всі елементи в один рядок
- вертикальне вирівнювання по центру
- акуратні однакові відступи між блоками
- summary і controls повинні виглядати як один компактний toolbar
- layout повинен відповідати зображенню-референсу

---

## Filters / Clear

У цій задачі треба залишити тільки:

```text
Clear ✕
```

Якщо активні filters відсутні або не повинні відображатися в цій конкретній реалізації - не додавати зайві pills.

---

## Devices summary

Формат:

```text
Devices: X | 🟢 x 🟡 x 🟣 x 🔴 x
```

Вимоги:
- компактне відображення
- кольори статусів мають залишатися узгодженими з іншими частинами UI
- summary повинен бути в одному блоці поруч з mode toggle

---

## Mode toggle

Відображення:

```text
[ End | All ]
```

Вимоги:
- segmented control
- розташований одразу після Devices summary
- активний стан чітко підсвічений

---

## Actions

Після mode toggle мають іти:

- `⇅` - sort
- `⟳` - refresh
- `Auto [ON]` - toggle автооновлення

---

## Вимоги до UX

- строка повинна читатися швидко
- основний акцент: Devices summary + mode toggle + actions
- не перевантажувати ліву частину

---

## Вимоги до тестування

Перевірити:

1. Усі елементи знаходяться в одному рядку
2. Порядок елементів відповідає макету
3. End / All працює як раніше
4. Sort / Refresh працюють як раніше
5. Auto toggle працює як раніше
6. Summary відображається коректно
7. Немає регресії в інших частинах сторінки

---

## Вимоги до документації

Якщо в проєкті є опис toolbar або UI layout:
- оновити коротко українською та англійською

Якщо окремої документації для цього немає - додаткову документацію не створювати.

---

## Критерії приймання

- строка виглядає як на референсному зображенні
- порядок елементів правильний
- layout компактний і чистий
- без зайвих змін поза межами цієї задачі

---

## Очікуваний результат

UI-строка виглядає так:

```text
Clear ✕   |   Devices: X | 🟢 x 🟡 x 🟣 x 🔴 x   [ End | All ]   ⇅   ⟳   Auto [ON]
```

і відповідає наданому макету.
