# TASK-077 - Restore active filters display and align toolbar (left filters, right controls)

## Контекст

Після останніх змін UI виникли дві проблеми:

1. Активні фільтри (наприклад: `OFFLINE`, `RANDOM`), обрані з таблиці, не відображаються у верхньому toolbar перед `Clear ✕`
2. Весь toolbar не розділений на логічні блоки:
   - ліва частина (filters + clear)
   - права частина (summary + mode + actions)

---

## Мета задачі

- повернути відображення активних фільтрів у toolbar
- зафіксувати правильну структуру:
  - filters + Clear зліва
  - всі інші елементи справа
- вирівняти layout відповідно до очікуваного UX

---

## Цільовий вигляд

```
[ OFFLINE ] [ RANDOM ]   Clear ✕                                  Devices: X | 🟢x 🟡x 🟣x 🔴x   [ End | All ]   ⇅   ⟳   Auto [ON]
```

---

## Що потрібно зробити

### 1. Відновити відображення активних фільтрів

Якщо користувач обрав фільтри (наприклад з рядків таблиці):

- `Status: offline`
- `Assignment: RANDOM`

вони повинні відображатися у вигляді badges перед `Clear ✕`

#### Вимоги:
- використовувати той самий badge компонент, що і в таблиці
- кольори повинні бути ті самі
- порядок: filters → Clear

---

### 2. Залишити `Clear ✕` у лівому блоці

`Clear ✕` повинен:

- знаходитись одразу після filters
- не зміщуватись вправо
- очищати всі активні filters

---

### 3. Розділити toolbar на два блоки

#### Лівий блок:
```
[ FILTERS ]   Clear ✕
```

#### Правий блок:
```
Devices: ...   [ End | All ]   ⇅   ⟳   Auto [ON]
```

---

### 4. Вирівнювання правого блоку вправо

Усі елементи правого блоку повинні:

- бути згруповані разом
- притиснуті до правого краю

---

### 5. Layout (обов’язково)

Реалізувати через flex:

- container:
  - display: flex
  - justify-content: space-between
  - align-items: center

- left group:
  - display: flex
  - gap

- right group:
  - display: flex
  - gap
  - align-items: center

---

### 6. Порядок елементів у правому блоці

Зліва направо:

1. `Devices: X | 🟢x 🟡x 🟣x 🔴x`
2. `[ End | All ]`
3. `⇅`
4. `⟳`
5. `Auto [ON]`

---

## Важливе правило

### Filters НЕ впливають на summary

- `Devices:` залежить тільки від Mode
- filters впливають тільки на таблицю

---

## Вимоги до тестування

### 1. Filters
- відображаються після вибору
- мають правильний стиль
- зникають після `Clear ✕`

---

### 2. Layout
- filters + Clear зліва
- всі інші елементи справа
- один рядок

---

### 3. Summary
- не змінюється при зміні filters
- змінюється при зміні Mode

---

### 4. Actions
- Sort працює
- Refresh працює
- Auto toggle працює

---

## Критерії приймання

- активні filters відображаються у toolbar
- `Clear ✕` знаходиться поруч з filters
- toolbar розділений на лівий і правий блок
- правий блок притиснутий вправо
- UI відповідає макету
- немає регресії

---

## Очікуваний результат

Toolbar працює так:

```
Filters → Clear (ліворуч)      |      Summary + Mode + Actions (праворуч)
```

і виглядає як у наданому референсі.
