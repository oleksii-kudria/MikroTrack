# TASK-007: Improve logging levels and add optional stdout output control

## Мета

Покращити систему логування та керування виводом результату:

- розділити INFO та DEBUG (DEBUG має бути значно детальнішим)
- прибрати дублювання логів
- додати керування виводом результату (JSON) у stdout через .env
- зробити поведінку передбачуваною для dev та production режимів

---

## Контекст

Зараз:

- INFO та DEBUG поводяться однаково
- JSON результат завжди виводиться у консоль
- немає можливості вимкнути stdout output

---

## Що потрібно реалізувати

### 1. Розділити рівні логування

#### INFO (мінімум)

- старт застосунку
- підключення до MikroTik
- успішне підключення
- отримання DHCP
- кількість записів
- завершення

---

#### DEBUG (деталізація)

- значення конфігурації (без password)
- кожен крок виконання
- виклики API
- кількість сирих записів
- приклади нормалізованих записів (1-2)
- процес нормалізації
- sanitized raw exception

---

### 2. Додати нову змінну в .env

PRINT_RESULT_TO_STDOUT=true

---

### 3. Оновити config.py

Додати параметр:

PRINT_RESULT_TO_STDOUT (bool)

---

### 4. Оновити main.py

if PRINT_RESULT_TO_STDOUT:
    print(json_result)

---

### 5. Поведінка системи

LOG_LEVEL=INFO → базові логи  
LOG_LEVEL=DEBUG → детальні логи  

PRINT_RESULT_TO_STDOUT=true → є JSON  
PRINT_RESULT_TO_STDOUT=false → тільки логи  

---

### 6. Безпека

- не логувати password
- використовувати sanitizer

---

## Очікуваний результат

INFO → короткі логи  
DEBUG → детальні  

stdout контрольований через env

---

## Критерії приймання

- INFO ≠ DEBUG
- JSON керується через env
- немає дублювання
- код простий
