# TASK-041 - Simplify session/state column and remove duplicate status indicators

## Контекст

Поточний стовпчик `Session / state time` містить:
- текстовий статус (`Online`, `Idle`, `Offline`)
- таймер
- додатковий рядок (`Last change`, `Idle since`, `Since`)

При цьому стан уже дублюється в першому стовпчику через кольоровий індикатор (зелений / жовтий / червоний / сірий).

Це створює:
- дублювання інформації
- перевантаження таблиці
- зайвий текст у кожному рядку
- дворядковий вигляд, який займає багато місця

## Мета

Спростити стовпчик `Session / state time`:
- залишити тільки таймер
- прибрати дублювання статусу
- перенести деталі в tooltip / hover

## Вимоги

### 1. Джерело truth для статусу

Статус пристрою відображається ТІЛЬКИ в першому стовпчику через кольоровий індикатор:

- green → online
- yellow/orange → idle
- red → offline
- gray → unknown

У стовпчику `Session / state time` текстові статуси:
- `Online`
- `Idle`
- `Offline`

БІЛЬШЕ НЕ ВІДОБРАЖАТИ.

---

### 2. Відображення таймера

У стовпчику `Session / state time` залишити тільки основний таймер.

Приклади:

```text
16:53:11
00:12
1 day ago
```

---

### 3. Hover / tooltip

При наведенні курсора на таймер показувати деталі:

#### Для online
```text
State: Online
Online since: 18:18
Last change: 18:18
```

#### Для idle
```text
State: Idle
Idle since: 22:52
Online since: 18:10
Last change: 22:52
```

#### Для offline
```text
State: Offline
Offline since: 21:38
Last change: 21:38
```

---

### 4. Прибрати другий рядок у колонці

Не відображати під таймером окремий другий рядок:
- `Last change: ...`
- `Idle since: ...`
- `Since: ...`

Вся ця інформація повинна бути доступна через hover.

---

### 5. UI стилі

Рекомендується:

- online timer → normal emphasis
- idle timer → slightly muted
- offline timer → muted / grayish

Але без дублювання кольорових статусів у самій колонці.

---

### 6. Optional: row background

Опціонально можна додати дуже легкий фон рядка:

- online → без фону
- idle → легкий теплий tint
- offline → легкий сірий / холодний tint

Але це НЕ обов’язково.
Головне — не перевантажити UI.

---

### 7. Acceptance Criteria

- текстові статуси `Online / Idle / Offline` у колонці відсутні
- другий рядок у колонці відсутній
- в колонці лишається тільки таймер
- статус видно тільки через кольоровий індикатор у першому стовпчику
- деталі доступні через hover
- таблиця виглядає компактніше та чистіше

---

## Очікуваний результат

Таблиця стає простішою:

- один індикатор статусу
- один таймер
- деталі за hover

Без дублювання логіки, без зайвого тексту, з кращим UX для оператора.
